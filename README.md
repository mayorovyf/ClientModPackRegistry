# ClientModPackRegistry

Локальный репозиторий правил для `ClientModPackToServer`.

Здесь лежит база знаний о модах: какие из них клиентские, какие можно оставить на сервере, какие ещё спорные и ждут проверки.

Этот репозиторий нужен для сборки `registry.bundle.json` и `manifest.json`, которые потом читает приложение. Внутри есть готовые production-правила, кандидаты из корпуса и отчётов, служебные скрипты и benchmark-данные. 

Это именно рабочий исходник registry, а не папка с готовыми серверными сборками.
